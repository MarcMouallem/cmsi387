Special Names & Symbols

The bash command-line shell attaches special meanings to certain names and symbols. Here is a list of a few. For the future complete documentation I might want to consider categorizing these.

Symbol Meaning
. Short hand for the path of the current directory.
. Precedes a filename, like so .my-file, to indicate the file is hidden.
~ Shorthand for the path to the userâ€™s home directory.
/ Separates directory names in a path.
\ Escape character, used to escape meanings of special characters. Commonly used to escape spaces.
.bash_profile A bash file, executed upon oping a login shell, that configures the terminal to certain user preferences.
$ Symbol used to prefix variable, as so $PATH, indicating the value of the variable is meant, not the literal string.
PATH Environmental variable that hold possible paths for commands entered without their full path.
SHELL Environmental variable which holds path of Bash application.
TERM I actually don't know this one.
USER Environmental variable which holds the user name of the current user.
HOME Environmental variable which holds the path to the current user's home directory.
LANG Environmental variable that holds the current language and character encoding.
